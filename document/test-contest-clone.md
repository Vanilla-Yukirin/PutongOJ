# 比赛克隆功能 — 测试用例

## 前置准备

- 1 个 Admin 账号
- 1 个非 admin 用户，在某课程中拥有 `manageContest` 角色（即课程教师）
- 1 个含若干题目的课程
- 1 个非课程比赛（admin 创建），含若干题目
- 1 个课程比赛，含若干题目

---

## 测试用例

| # | 操作用户 | 操作 | 预期结果 |
|---|---------|------|---------|
| T1 | Admin | 打开任意比赛 → Edit 标签 → 点击"克隆比赛" | 对话框打开，标题预填为"Clone of X"，isHidden/isPublic 从来源复制 |
| T2 | Admin | 克隆一个非课程比赛，修改标题+日期，提交 | 新比赛创建成功，跳转至其 Edit 页，题目/允许语言/标签样式与来源一致 |
| T3 | Admin | 克隆一个课程比赛 | 新比赛在同一课程下创建，所有题目被复制 |
| T4 | 课程教师 | 打开自己课程内的比赛 → Edit → 克隆 | 对话框打开，课程显示为只读 |
| T5 | 课程教师 | 提交克隆对话框 | 在同一课程下成功创建新比赛 |
| T6 | 课程教师 | 尝试访问自己没有 jury 权限的比赛的 Edit 标签 | Tab 不显示 → 克隆按钮不可达（已有鉴权） |
| T7 | Admin | 克隆一个 0 道题目的比赛 | 新比赛创建成功，题目列表为空，无报错 |
| T8 | Admin | 克隆对话框：标题留空 | HTML5 `required` 阻止提交 |
| T9 | Admin | 克隆对话框：结束时间早于开始时间 | 校验提示"结束时间必须晚于开始时间" |
| T10 | Admin | 提交克隆时服务器不可用 | 显示错误 toast，对话框保持打开 |

---

## T2 详细步骤（Happy Path）

1. 以 Admin 登录
2. 进入比赛列表 → 找到一个非课程比赛 → 点击标题进入 → 切换到 Edit 标签
3. 滚动至页面底部 → 点击"克隆比赛"
4. 对话框打开：标题 = "Clone of [原标题]"，日期和开关已预填
5. 将标题改为"Test Clone"，调整日期 → 点击克隆
6. 浏览器跳转至新比赛的 Edit 页面
7. 验证：标题、题目、允许语言、标签样式与来源一致；课程 = 无
8. 验证：startsAt/endsAt 与对话框中填写的时间一致
